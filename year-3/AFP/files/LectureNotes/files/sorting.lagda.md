# A Specification of Sorting Algormithms

## Sorted Lists

Now that we have a notion of a [partial
order](partial-orders.lagda.md), we can say what we mean for a list to
be sorted.  We do so using the following inductively defined
predicate.

```agda
module _ {X : Type} (ρ : PartialOrder X) where
  open PartialOrder ρ 

  data Sorted : List X → Type where
    nil-sorted : Sorted []
    sing-sorted : (x : X) → Sorted (x :: [])
    adj-sorted : {x y : X} (xs : List X) 
      → x ≤ y
      → Sorted (y :: xs) 
      → Sorted (x :: y :: xs)
```

The various constructors express the following ideas:

1. The empty list is always sorted
1. A list containing a single elements is sorted
1. When adjoining two elements `x` and `y` to a list `xs`, if we know that `x ≤ y` and
we know that the list `y :: xs` is itself sorted, then the list `x :: y :: xs` is also sorted.

As an example, here is what the proof that the list `1 :: 2 :: 3 :: []`
is a sorted list for the usual ordering on `ℕ` looks like:

```agda
ex₀ : Sorted ℕ-PartialOrder (1 :: 2 :: 3 :: [])
ex₀ = adj-sorted (3 :: []) (suc-preserves-≤ 0-smallest)
       (adj-sorted [] (suc-preserves-≤ (suc-preserves-≤ 0-smallest))
        (sing-sorted 3)) 
```

Next, we'll try to say what a *sorting algorithm* is.  The idea is
that it should consist of a function `σ : List X → List X` which "does
the sorting" together with a proof that for any list `xs : List X`
applying the function `σ` results in a sorted list.  We can axiomatize
this as follows:

```agda
SortingAlgorithm-naive : {X : Type} (ρ : PartialOrder X) → Type
SortingAlgorithm-naive {X} ρ =
  Σ σ ꞉ (List X → List X) ,
  ((xs : List X) → Sorted ρ (σ xs)) 
```

Unfortunately, a bit too naive, as it does not exclude the 
trivial algorithm which simply always returns an empty list:

```
TrivialSort : {X : Type} (ρ : PartialOrder X) → SortingAlgorithm-naive ρ
TrivialSort p =  (λ xs → []) , λ xs → nil-sorted
```

We clearly need some way to exclude this kind of trivial example.

## Permutations

What is missing from our previous specification is the statement that
**the sorted list should have the same elements as the input list**.
Our next task, therefore is to make this idea precise.  Let us begin
by defining the *positions* of a list by recursion:

```agda
Pos : {X : Type} → List X → Type
Pos [] = 𝟘
Pos (_ :: xs) = 𝟙 ∔ Pos xs
```
The intuition is that the positions are the valid indices of the list,
but rendered as a type.  Each time the list is extended by one, there
is a new position, namely the one in which the new element sits.

Once we have the notion of position, we can write a function which
extracts the element at that position:
```agda
_!!_ : {X : Type} (xs : List X) → Pos xs → X
(x :: xs) !! inl ∙ = x
(x :: xs) !! inr p = xs !! p
```
To say what this function does in words: when the position is the new
one generated by appending an element, we simply return that element.
Otherwise we just recurse.

Now we can say what it means for one list to be a *permutation* of
another.  First, we require an isomorphism between the sets of
positions.  This ensures, for example, that the lists have the same
length.  (**Exercise:** Prove this!).  Then we ask that for every
position, the inhabitant of the first list at that position is equal
to the inhabitant of the second list at the position specified by the
given isomorphism.

```agda
record _is-permutation-of_ {X : Type} (xs ys : List X) : Type where
  field
    pos-iso : Pos xs ≅ Pos ys
    same-el : (p : Pos xs) → xs !! p ≡ ys !! bijection pos-iso p 
```

##  Sorting Algorithms

Now that we have the notion of permutation, we can improve our specification
of sorting algorithms.  We will formulate the spec as a record:

```agda
record SortingAlgorithm {X : Type} (ρ : PartialOrder X) : Type where
  field
    sort : List X → List X
    sort-is-sorted : (xs : List X) → Sorted ρ (sort xs)
    sort-is-permutation : (xs : List X) → (sort xs) is-permutation-of xs
```
